Toolchain to support compiling with gcc, clang, and other compilers using a compatible command line interface.

Values Added to projectSettings:

This toolchain will add the following items to the csbuild.projectSettings.projectSettings class when it is the active toolchain:

  • warn_flags: list[str] containing warn flags
  • cppstandard: str containing the C++ standard to use
  • cstandard: str containing the C standard to use
  • strictOrdering: bool determining whether or not to require strict order of libraries when linking

EnableStrictOrdering(self)

source code 

By default, csbuild uses --start-group and --end-group to eliminate GCC's requirements of strictly managed link order. This comes with a performance cost when linking, however, so if you would prefer to manage your link order manually, this function will disable csbuild's default --start-group/--end-group behavior.

DisableStrictOrdering(self)

source code 

Use --start-group/--end-group to eliminate the need to strictly order libraries when linking. This is the default behavior.
